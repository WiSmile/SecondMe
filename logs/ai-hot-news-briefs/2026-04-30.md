## 插件 / skill / agent 工具观察

- 事实：OpenAI 的 Agents SDK 已把代码执行、浏览器和 computer use 打包成更统一的 agent 运行层；Anthropic 则继续把 Claude 向设计工具、创意软件和技能体系延展。
- 推断：`plugin -> skill -> agent runtime` 正在快速收敛成同一条产品链路，差异不再只是“接了哪些工具”，而是“谁能把权限、上下文、执行环境和沉淀机制统一起来”。
- 建议：SecondMe 当前最该继续强化的不是新插件数量，而是三件事：
  1. 让 skill、工作流、日志留痕继续保持同一套可检索结构。
  2. 把执行边界和审计记录继续做成默认能力，而不是额外步骤。
  3. 观察是否需要把“热点信号 -> 产品判断 -> 需求池结果”做成更模板化的 agent 流程。
