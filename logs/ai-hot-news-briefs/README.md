# AI 热点早报留痕

这里用于保存 AI 热点早报的最小可检索留痕，供总经办、产品战略部和每日复盘读取。

## 目标

- 让早报不只存在于聊天会话里
- 让产品战略部有固定入口消费早报
- 让每日复盘能明确判断“已消费 / 未消费 / 不可消费”

## 文件命名

- 推荐格式：`YYYY-MM-DD.md`
- 如果同一天有多个版本，可使用 `YYYY-MM-DD-a.md`、`YYYY-MM-DD-b.md`

## 最小模板

```md
# YYYY-MM-DD AI 热点早报

## 日期

2026-04-27

## 会话标题

2026-04-27 AI 热点早报

## 核心热点

1. 热点一
   - 意义判断：
   - 与 Vince / SecondMe 的相关性：
2. 热点二
   - 意义判断：
   - 与 Vince / SecondMe 的相关性：
3. 热点三
   - 意义判断：
   - 与 Vince / SecondMe 的相关性：

## 产品战略部处理结果

- 进入需求池：PS-xxxx
或
- 仅观察：原因
或
- 当前阶段无关：原因

## 备注

- 如果早报会话已产生，但当天还没完成产品战略消费，也要先留一份摘要索引，并明确标注“联动未完成”。
```

## 读取顺序

产品战略部和每日复盘默认按以下顺序读取：

1. 当天 `YYYY-MM-DD.md`
2. 当天其他变体文件
3. 当前任务上下文中直接可见的早报内容

如果三者都没有，就应标注为“当前不可消费”。
