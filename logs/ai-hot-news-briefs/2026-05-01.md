## 插件 / skill / agent 工具观察

- 推断：`plugin -> skill -> agent runtime -> audit` 正在收敛为同一层产品能力，未来差异不只在模型分数，而在上下文管理、权限治理和执行闭环。
- 建议：SecondMe 现阶段最值得继续强化三件事：
  1. 保持 skill、workflow、log 三者使用同一套可检索结构。
  2. 把事实 / 推断 / 建议继续固定成默认输出格式。
  3. 继续观察是否需要把“热点信号 -> 产品判断 -> 需求池结果”进一步模板化成 agent 流程。
