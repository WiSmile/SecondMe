# 每日复盘工作流

## 目标

每日复盘由总经办发起、产品战略部主责产品分析，用于让 SecondMe 持续理解 Vince 和 Vincy 的协作方式，并把当天对话中出现的系统改进点转化为可追踪的优化记录和迭代建议。

默认执行时间为 Asia/Shanghai 每天 22:30。

## 输入范围

每日复盘读取以下信息：

- `SecondMe` 工作区内当天可见的 Vincy 相关对话、任务记录和仓库变更
- 工作区之外与 Vincy 明确相关的对话信息
- `logs/` 中当天运行记录、决策记录和变更记录
- `06-behavior-lab/` 中已有画像观察
- `08-shared-memory/` 中当前共享记忆和记忆写入规则
- `09-agent-factory/` 与 `10-harness/` 中和 agent 演进、执行边界相关的文件
- AI 热点早报、趋势摘要或当日可见的外部机会信号
- `logs/ai-hot-news-briefs/` 中当日或最近一次可见的早报留痕

如果某类输入不可见，应在日报中说明缺口，不把缺失信息当作事实。
如果某类输入已经产生，但尚未形成可检索留痕或消费结果，也应明确标注为“已产生但未完成联动”，而不是直接视作不存在。

## 主责分工

- 总经办：发起复盘、收集输入、协调部门、向 Vince 发送日报
- 产品战略部：主责产品判断、需求池维护、优先级和启动建议
- 行为研究部：判断画像变化是否属于稳定偏好或仅为待确认观察
- Agent Factory：判断哪些建议涉及 Vincy / SecondMe 的角色和提示词演进
- Harness：判断哪些建议会碰权限、分支、审计或执行边界

## 输出

每日复盘输出三类内容：

1. 画像迭代
   - Vince 的沟通偏好、决策偏好、工作节奏、稳定目标
   - Vincy 的表达风格、协作方式、需要保持或修正的互动习惯
   - 每条观察应区分事实、推断、待确认观察和已确认偏好

2. SecondMe 自我更新
   - 提炼当天 Vince 提出的改进点
   - 判断哪些属于低风险记录更新，哪些属于重要文档、SOP、agent 提示词或权限规则变更
   - 对需要确认的事项生成清晰草案，不直接改动核心规则

3. 产品战略输出
   - 给出今日产品判断
   - 维护“迭代规划建议需求池”的新增项和状态变化
   - 为值得保留但暂不启动的建议标注优先级、启动时机和触发条件
- 如果当天存在 AI 热点早报，明确其是否已被产品战略部消费，以及消费结果
   - 优先消费仓库内 `logs/ai-hot-news-briefs/` 的当日留痕；若只有会话存在、没有留痕，也要明确标注

4. 日报通知
   - 在单独的日报线程发送摘要
   - 列出已自动更新、待确认草案、风险提醒和次日建议
   - 说明是否产生 git 提交、候选分支或 AB 测试建议

## 更新分级

每日复盘的 A / B / C 档是对 SecondMe A0 / A1 / A2 权限模型的场景化映射；权限边界仍以 `00-governance/permission-boundaries.md` 为准。

### A 档：可自动落地

低风险、可逆、偏记录型内容可以自动更新：

- 向 `06-behavior-lab/profile-observations.md` 追加画像观察
- 向 `logs/run-log.md` 追加每日复盘执行记录
- 向 `logs/change-log.md` 追加已执行的低风险文档变更
- 向 `09-agent-factory/vincy-self-improvement-log.md` 追加自我优化观察
- 创建或追加每日复盘报告索引

### B 档：生成草案，等待确认

影响重要协作规则或长期行为的内容，只生成草案和建议：

- 修改 agent 岗位说明书
- 修改 Vincy 入口说明或核心身份描述
- 更新部门 SOP
- 修改共享记忆中的稳定长期事实
- 调整日报工作流本身

### C 档：只报告，不执行

高风险、外部影响或核心权限相关动作只报告，不自动执行：

- 删除或覆盖重要内容
- 修改权限边界
- 修改核心 agent 身份设定
- 对外发送消息、邮件、简历或申请
- 执行电脑操作或接入新的高权限工具

## Git 策略

- A 档低风险、已确认或偏记录型更新可以直接提交到 `master`。
- B 档待确认草案应提交到 `pre-release`，或只在日报中列为待确认项。
- 涉及表达风格、流程口径、agent 行为差异且存在多个候选方案时，优先创建 `codex/ab-<topic>-<variant>` 形式的候选分支。
- 日报中应标明当天是否产生提交、分支或待确认草案。
- 当有两种以上可行口径时，可以使用 AB 测试方式保留两个候选分支，并在日报中说明差异和推荐选择。
- 具体分支规则见 `10-harness/branch-management.md`。

## 日报格式

日报建议包含以下部分：

- 今日对话摘要
- Vince 画像更新
- Vincy 画像更新
- SecondMe 自我优化
- 今日产品判断
- 需求池新增 / 状态变化
- AI 热点早报消费状态
- 已自动落地
- 待 Vince 确认
- 风险与缺口
- 明日建议

## 标题规范

- 每日复盘会话标题应包含绝对日期。
- 推荐格式：`YYYY-MM-DD SecondMe 每日复盘`
- 示例：`2026-04-26 SecondMe 每日复盘`

## 审计要求

每日复盘应遵守 `00-governance/permission-boundaries.md` 和 `08-shared-memory/memory-rules.md`。

不确定的信息应放入待确认观察，不应写成稳定事实。
