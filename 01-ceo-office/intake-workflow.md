# 任务接收和分发流程

## 入口

所有新资料、想法、链接、笔记和待处理任务优先进入：

- `inbox/`
- 或直接告诉 Chief of Staff Agent

## 分发规则

| 内容类型 | 负责部门 |
| --- | --- |
| 学习笔记、课程内容 | 学习发展部、知识管理部 |
| 简历、岗位 JD、面试题 | 职业发展部 |
| PHP、项目、脚本、开发问题 | 研发工程部 |
| 习惯、偏好、作息、沟通风格 | 行为研究部 |
| 权限、审批、敏感操作 | 治理审计部 |
| 自动化、工具、电脑操作流程 | 运营与 harness |

## 标准处理步骤

1. 识别资料类型
2. 判断是否涉及权限风险
3. 分发给对应部门
4. 产出整理结果或行动建议
5. 写入共享记忆或对应知识库
6. 必要时更新日志
