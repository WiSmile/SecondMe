# 需求说明：SecondMe 项目展示页

## 来源

- 来源类型：Vince 直接提出
- 原始意图：做一个 HTML 页面，展示目前的 SecondMe 项目
- 需求归档位置：产品战略部 `11-product-strategy/requirements/`

## 背景

SecondMe 当前是一个文档驱动的个人 AI 公司操作系统，包含组织结构、Vincy 入口、部门职责、治理边界、共享记忆、Harness 执行层和产品战略部需求池。

现有内容主要以 Markdown 文件存在，适合内部运行，但不适合让潜在合作者或面试官快速理解项目价值。因此需要一个高质量 HTML 展示页，把 SecondMe 的定位、结构和可信边界整理成可浏览页面。

## 目标受众

- Vince 自己：用于复盘项目状态、切换上下文、展示当前成果
- 潜在合作者：快速理解 SecondMe 的工作方式和协作边界
- 面试官：看到项目的产品思考、工程组织能力和 AI 协作方法论

## 核心目标

1. 用中文清晰解释 SecondMe 是什么，保留 `SecondMe` 英文公司名。
2. 让外部读者在首屏理解：SecondMe 不是聊天机器人，而是 Vince 的个人 AI 公司。
3. 展示 SecondMe 的组织结构、部门职责、权限治理和路线图。
4. 将 Dashboard 作为菜单中的内部视图，展示部门、状态、记忆、日志和迭代信息。
5. 页面视觉要高级、克制、有作品集质感，避免廉价模板感。

## 范围

本需求包含：

- 单个可直接打开的 HTML 页面
- 内联 CSS 和少量必要交互
- 中文为主的页面文案
- Dashboard 中英文双语部门名称
- 基于当前仓库文档的项目内容提炼

本需求暂不包含：

- 后端服务
- 构建系统
- 外部部署
- 自动读取 Markdown 并生成页面
- 真实登录、权限系统或动态数据

## 验收标准

- 页面可以在浏览器中直接打开并完整显示。
- 首屏明确出现 `SecondMe`，并传达“个人 AI 公司”定位。
- 页面导航至少包含项目概览、组织结构、治理边界、路线图和 Dashboard。
- Dashboard 中的部门名称采用中英文双语，例如“总经办 CEO Office”。
- 页面默认中文，`SecondMe` 不翻译。
- 视觉风格高级、清晰、适合给潜在合作者或面试官查看。
- 移动端和桌面端没有明显文字重叠或布局错乱。
