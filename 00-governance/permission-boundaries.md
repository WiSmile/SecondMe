# 权限边界

SecondMe 采用三层权限模型：自动、确认后执行、必须批准。

## 可以自动执行

低风险、可逆、以整理和记录为主的动作可以自动执行：

- 学习内容整理
- 知识沉淀和摘要
- 标签建议和标签更新
- 学习状态更新
- 个人画像草稿更新
- 非破坏性索引和归档建议

自动执行后应在相关日志中记录来源、时间、动作和结果。

## 需要我确认后执行

会影响重要资料、工作成果或长期规则的动作，需要先给出方案并等待我确认：

- 写代码
- 修改文档正文
- 整理重要资料
- 更新部门 SOP
- 更新 agent 提示词草稿
- 调整知识库分类体系
- 生成对外使用的简历、邮件、作品集文案

## 必须我批准

高风险、外部影响、破坏性或核心身份相关动作，必须明确获得我的批准：

- 电脑操作
- 对外发送消息、邮件、简历或申请
- 删除或覆盖重要内容
- 修改权限规则
- 修改核心 agent 身份设定
- 接入新的高权限工具
- 批量移动、重命名或清理重要资料

## 默认保守规则

当无法判断一个动作属于哪一层时，按更高权限层级处理。
