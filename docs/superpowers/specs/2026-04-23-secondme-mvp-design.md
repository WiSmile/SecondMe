# SecondMe MVP Design

## Goal

Create a documentation-driven MVP for SecondMe, a personal AI company that supports long-term learning, knowledge capture, job search growth, development collaboration, and future controlled computer operation.

## Approach

The first version uses Markdown files and directories as the system of record. This keeps the project transparent, editable, auditable, and immediately useful before adding automation.

## Scope

Included:

- Governance documents
- Department directories
- Agent job descriptions
- Shared memory documents
- Inbox workflow
- Roadmap and first-stage plan
- Harness module boundaries

Deferred:

- Automated dispatcher
- Script-based memory indexing
- Computer operation
- External sending
- Agent self-modification

## Permission Model

SecondMe uses three levels:

- A0: automatic low-risk organization and memory updates
- A1: confirmation required before changing important docs, code, SOPs, or prompts
- A2: explicit approval required for computer operation, external sending, deletion, permission changes, and core identity changes

## First-Stage Success Criteria

- New information has a single entry point
- Tasks can be routed through the CEO office
- Knowledge can be summarized, tagged, and archived
- Job-search materials can improve over time
- Personal preferences and context can be stored carefully
- Important changes are logged
