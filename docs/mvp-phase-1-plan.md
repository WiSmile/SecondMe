# 第一阶段最小可用版本落地计划

## 目标

建立一个能真正帮助我学习沉淀、求职成长和积累个人画像的最小闭环。

## 成功标准

- 新资料可以进入 `inbox/`
- 总经办可以判断资料去向
- 知识管理部可以完成摘要、标签和归档
- 共享记忆能记录当前目标、偏好和重要事实
- 求职资料能进入职业发展部持续优化
- 所有重要变更有日志记录

## 第 1 步：初始化组织结构

已创建：

- 治理审计部
- 总经办
- 知识管理部
- 共享记忆
- 其他部门占位

## 第 2 步：建立资料流转闭环

流程：

1. 新资料进入 `inbox/`
2. Chief of Staff Agent 判断类型
3. Archivist Agent 整理和打标签
4. 重要信息写入 `08-shared-memory/`
5. 求职内容同步到 `04-career/`
6. 变更记录写入 `logs/`

## 第 3 步：服务当前求职

优先维护：

- `04-career/resume-notes.md`
- `04-career/job-analysis-template.md`
- `04-career/interview-prep-template.md`

## 第 4 步：建立学习复盘节奏

优先维护：

- `02-learning/learning-plan-template.md`
- `02-learning/review-template.md`
- `03-knowledge/knowledge-entry-template.md`

## 第 5 步：每周复盘

每周由 Chief of Staff Agent 汇总：

- 学习进展
- 求职进展
- 知识库更新
- 个人画像观察
- 下周建议
